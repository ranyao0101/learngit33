# simple-todo_01
一个纯文本记录的简易待办事项清单项目，用于实践GitHub仓库管理、分支操作。


## 1. 项目结构
项目仅包含3个核心文件：
- `todo.txt`：待办事项的纯文本记录文件
- `README.md`：项目说明文档
- `LICENSE`：MIT开源许可证文件


## 2. 分支管理实践（核心体现）
本项目通过以下步骤完成分支管理、合并、冲突解决：

### （1）创建分支
基于默认的`main`分支，创建2个功能分支：
- `feature/add-todo`：用于新增待办事项
- `feature/edit-todo`：用于修改已有待办事项


### （2）分支内容提交
- 在`feature/add-todo`分支中，编辑`todo.txt`，添加内容：完成了学习任务

提交并推送到远程仓库。

- 在`feature/edit-todo`分支中，编辑`todo.txt`（**故意修改和`feature/add-todo`重叠的内容**）：修改一次

提交并推送到远程仓库。


### （3）合并分支 & 解决冲突
1. 先合并`feature/add-todo`到`main`：
 - 切换到`main`分支，执行合并操作（无冲突，直接合并成功）。

2. 再合并`feature/edit-todo`到`main`：
 - 此时`todo.txt`中“1. 完成GitHub仓库创建”这一行，在两个分支中内容不一致，触发**合并冲突**。
 - 手动解决冲突：打开`todo.txt`，将内容修改为最终版本：最终版

 - 提交冲突解决后的内容，完成合并。

- 看机会：可扩展为多格式（Markdown）待办工具

### 三定：
- 定战略控制点：保持“纯文本、无依赖”的极简定位
- 定目标：完成GitHub仓库管理全流程实践
